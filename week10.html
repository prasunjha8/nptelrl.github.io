Hierarchical
Reinforcement Learning

Scaling RL to large environments through temporal abstraction (macro-actions), state abstraction, and reusable sub-policies: SMDPs, Options framework, and HAMs.

Section 01

Introduction to Hierarchical RL

Humans don't plan every footstep when navigating to a train station. We decompose: "Exit building" → "Walk to bus stop" → "Take bus." HRL replicates this by breaking the MDP into a hierarchy of sub-problems.

Key Advantages

  • Temporal Abstraction: "Exit building" spans many steps but the planner treats it as one action
  • State Abstraction: Solving "exit building" doesn't require knowing train timetables — irrelevant variables dropped
  • Reusability / Transfer: The "exit building" policy works whether the goal is the station, cafeteria, or library
Assignment 10 · Q6

Which are possible advantages of formulating a problem as a hierarchy of sub-problems?
(a) Reduced state space. (b) Meaningful state abstraction. (c) Temporal abstraction. (d) Reusability.

Solution
✓ All of (a), (b), (c), (d)

All four are genuine advantages of hierarchical decomposition.

Section 02

Semi-Markov Decision Processes (SMDPs)

Standard MDPs assume every action takes exactly 1 step. Macro-actions take variable time $\tau$ — we need SMDPs to model this.

SMDP Q-Learning Update

The agent accumulates discounted rewards over $\tau$ steps, and the next-state value is discounted by $\gamma^\tau$:

SMDP Q-Learning
$$Q(s,a) \leftarrow Q(s,a) + \alpha\!\left[\underbrace{r_t + \gamma r_{t+1} + \cdots + \gamma^{\tau-1}r_{t+\tau-1}}_{A} + \underbrace{\gamma^\tau}_{B}\!\max_{a'} Q(s', a') - Q(s,a)\right]$$
Assignment 10 · Q1

In the SMDP Q-learning update $Q(s,a) = Q(s,a) + \alpha[A + B\max_{a'}Q(s',a') - Q(s,a)]$, what are $A$ and $B$?

Solution
✓ (b) — A = sum of discounted rewards during τ steps; B = γⁿ

$A$ accumulates actual rewards during the macro-action's execution. $B = \gamma^\tau$ discounts the next state by the full duration of the macro-action.

Bellman Equation for SMDPs

Optimal SMDP Bellman Equation
$$V^*(s) = \max_{a \in A(s)} \left[\sum_{s',\tau} P(s',\tau \mid s,a)\left(R(s,a,\tau,s') + \gamma^\tau V^*(s')\right)\right]$$
Assignment 10 · Q10

Correct Bellman equation for an SMDP?

Solution
✓ (c) — The formula above

Must sum over the joint distribution of next state $s'$ and duration $\tau$. The discount $\gamma^\tau$ reflects how many actual time steps elapsed.

SMDP vs Conventional Q-Learning

Assignment 10 · Q2

If next state and reward depend only on $(s,a)$ (not $\tau$), does conventional Q-learning yield the same policy as SMDP Q-learning?

Solution
✓ No

Even with state/reward independent of $\tau$, the $\gamma^\tau$ factor changes relative weighting of short vs long macro-actions, potentially altering the optimal policy.

Assignment 10 · Q7

If $\tau$ is fixed for all state-action pairs, do conventional and SMDP Q-learning give the same policy for $\tau=3$, $\tau=2$, $\tau=1$?

Solution
✓ (d) — No, No, Yes

Only when $\tau=1$ does SMDP collapse to standard MDP (discount $\gamma^1 = \gamma$). For $\tau > 1$, the $\gamma^\tau$ scaling changes the reward weighting, potentially changing the optimal policy.

Assignment 10 · Q9

Which scenarios satisfy $P(s',\tau \mid s,a) = P(s' \mid s,a)P(\tau \mid s,a)$ (next state and transition time are independent)?

Solution
✓ (a) Single door; (c) Primitive actions only

(a) One door: time to exit depends only on start position, not on which door you exit (only one option). (c) Primitive actions take $\tau=1$ deterministically — trivially independent. (b) Two doors: the door you exit through (next state) directly determines how long it took — dependent.

Section 04

Hierarchies of Abstract Machines (HAMs)

HAMs constrain the agent to execute programs defined by finite state machines. Four types of machine states:

State TypeWhat it doesRL agent decides here?
Action StateEmits a primitive action to the environmentNo — hardcoded
Call StateSuspends current machine, calls a sub-machineNo
Choice StateFork — agent chooses which machine state to go to nextYes
Stop StateTerminates current machine, returns to callerNo
💡 The RL agent only acts at Choice States. All other transitions are deterministic machine logic.
Practice Assignment 10 · Q3

Which are true for HAMs?
(a) Machines are connected as a DAG.
(d) Action states are the only ones that interact with the environment.

Solution
✓ (a), (d)

(a) Must be a DAG to prevent infinite recursive loops. (d) Only action states emit primitive actions — other states transition within or between machines. Choice states select within the same machine; stop states return to the parent machine.

Assignment 10 · Q5

Statement 1: SMDP state = base MDP state + call stack + current machine state.
Statement 2: SMDP actions are defined by action states.

Solution
✓ (b) — Statement 1 True, Statement 2 False

The full SMDP state includes where we are in the machine stack (S1 true). But RL decisions are made at choice states, not action states — action states are hardcoded (S2 false).

Practice Assignment 10 · Q4

In a HAM, the core MDP state changes only when we visit a choice state. True or False?

Solution
✓ False

The environment state changes only when a primitive action is executed — which happens exclusively at action states, not choice states.

Assignment 10 · Q3

What is the immediate reward received between two choice states in a HAM?

Solution
✓ (a) — Accumulated sum of all core MDP rewards between the two choice points

The agent decides at choice state A; the machine runs through action states until the next choice state B. The reward for A's decision is the total sum of all primitive rewards during that execution.

Section 05

Types of Optimality in HRL

Hierarchical constraints restrict the policy space, leading to three different notions of optimality:

Flat Optimal (F)
Ignores hierarchy entirely. Best possible policy in the base MDP.
Highest return. Hardest to learn.
Hierarchically Optimal (H)
Best policy within hierarchical graph constraints.
Return ≤ Flat. Easier to learn.
Recursively Optimal (R)
Every sub-task is locally optimal, regardless of global effect.
Lowest return. Easiest to learn.
Ordering of Expected Returns
$$R \;\leq\; H \;\leq\; F$$
Practice Assignment 10 · Q1

Expected ordering of total reward for hierarchically optimal (H), recursively optimal (R), and flat optimal (F)?

Solution
✓ (c) — R ≤ H ≤ F

Flat has full policy freedom — maximum return. Hierarchical is constrained by the graph. Recursive is additionally constrained by forced local sub-task optimality — most restrictive, lowest potential return.

Practice Assignment 10 · Q2

Do flat optimal solutions give the highest expected return?

Solution
✓ Yes

Flat optimality is unconstrained — it searches the full policy space and always finds the global maximum.

Practice Assignment 10 · Q5

Flat optimal policies are always easier to learn than recursively optimal policies. True or False?

Solution
✓ False

Flat policies yield better returns but are significantly harder to learn — they face the full curse of dimensionality. Recursive policies work in heavily reduced sub-task state spaces and benefit from reuse, making them much easier to learn despite lower potential returns.
