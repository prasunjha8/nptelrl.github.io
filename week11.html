MAXQ & Option
Discovery

MAXQ decomposes the value function itself across a hierarchy of sub-tasks. Option Discovery automates finding those sub-tasks using structural analysis of the state space.

Section 01

The MAXQ Framework & Sub-tasks

Where Options and HAMs constrain the policy, MAXQ decomposes the value function itself. The core MDP is split into a hierarchy of sub-tasks $M_0, M_1, \dots, M_n$, where $M_0$ is the root (solving the entire problem).

Sub-tasks as SMDPs

Each sub-task $M_i$ can invoke both primitive actions (1 step) and other sub-tasks (variable $\tau$ steps). Because transitions occur over variable time intervals, every sub-task must be modelled as an SMDP.

Practice Assignment 11 · Q1

Each sub-task $M_i$ in MAXQ is not an SMDP. True or False?

Solution
✓ False

Because sub-tasks can invoke other sub-tasks (variable-duration macro-actions), transitions span varying time intervals — the defining characteristic of an SMDP. Every sub-task is an SMDP.

Sub-task Termination

MAXQ partitions all MDP states into active states and terminal states for each sub-task. Sub-task $M_i$ terminates immediately and deterministically when the agent enters a terminal state for $M_i$.

Assignment 11 · Q1

How does sub-task termination work in MAXQ?

Solution
✓ (b) — Termination is deterministic

States are partitioned into active and terminal sets per sub-task. The moment the agent enters a terminal state for the current sub-task, it terminates — no probability involved.

Section 02

Call Graphs & Taxi World

Sub-task dependencies are shown in a Call Graph where nodes are tasks and directed edges show which tasks can invoke which.

Practice Assignment 11 · Q2

What structural property must the MAXQ call graph satisfy?

Solution
✓ (b) — Must be a DAG (Directed Acyclic Graph)

Cycles would cause infinite loops (A calls B, B calls A) without ever executing a primitive action. A DAG guarantees that execution always terminates at leaf nodes (primitives).

Assignment 11 · Q5

Is the left-to-right ordering in a MAXQ call graph important for execution order?

Solution
✓ False

Visual layout is purely for readability. The policy dynamically determines which sub-task to call based on the current state — not the diagram's left-to-right ordering.

Assignment 11 · Q3

From: (i) Left, (ii) Drop-off, (iii) Navigate, (iv) put-down — which are primitive actions?

Solution
✓ (c) — (i) Left, (iv) put-down

Primitive actions are single-step, native to the environment: directional moves and pick/put. Drop-off and Navigate are abstract sub-tasks composed of sequences of primitives.

Parameterising Navigate with a destination allows the same learned navigation logic to be reused for all four possible target locations.

Section 03

MAXQ Value Function Decomposition

The key insight: the value of the root task can be computed by summing values of active sub-tasks down the hierarchy.

Value Function Notation: $V^\pi(i, s)$

In hierarchical RL, value depends not just on state but on what you are currently trying to accomplish. Being at position (2,2) has a different value if you are in the Pick-up sub-task versus Drop-off.

Assignment 11 · Q2

In $V^\pi(i, s)$, what is $i$?

Solution
✓ (c) — Identity of the sub-task / SMDP being executed

$i$ tells us the hierarchical context. The same state $s$ has different values depending on which sub-task the agent is currently executing.

The Decomposition Equation

For sub-task $i$ choosing to execute action $a$:

MAXQ Q-Value Decomposition
$$Q^\pi(i, s, a) = V^\pi(a, s) + C^\pi(i, s, a)$$
Qπ(i, s, a)
Total action-value of taking action $a$ in state $s$ under sub-task $i$
=
Vπ(a, s)
Return accumulated during execution of action $a$
+
Cπ(i, s, a)
Completion: return accumulated after $a$ finishes, to complete sub-task $i$
💡 $V^\pi(a, s)$ is the value function of the child sub-task $a$ (what happens during $a$'s execution). $C^\pi(i, s, a)$ captures what the parent sub-task $i$ still needs to do after $a$ completes.
Assignment 11 · Q9

Taking action $a$ gives reward $r_1$; after $a$ completes, reward $r_2$ finishes sub-task $i$. What is $C^\pi(i, s, a)$?

Solution
✓ (a) — $C^\pi(i, s, a) = r_2$

$V^\pi(a, s)$ captures $r_1$ (during $a$). The completion function captures what remains after $a$ finishes — which is $r_2$.

Practice Assignment 11 · Q4

Is $\bar{R}(s,a)$ (expected reward of SMDP for $M_i$) equivalent to $V^{\pi_i}(a,s)$?

Solution
✓ False

The accumulated reward during $a$ evaluates to $V^\pi(a,s)$ (global hierarchical policy $\pi$), not $V^{\pi_i}(a,s)$ (isolated sub-task policy $\pi_i$). The notation matters — the global policy is applied, not the sub-task-local one.

Section 04

Learning with Pseudo-Rewards

The environment doesn't reward the agent for reaching a doorway — it only rewards the ultimate task goal. To train sub-tasks that target intermediate states (like "reach the door"), we supply artificially designed pseudo-rewards.

Practice Assignment 11 · Q3

In MAXQ, the agent is restricted to the sub-task's pseudo-rewards (core MDP rewards unavailable while learning sub-tasks). True or False?

Solution
✓ False

Core environment rewards are always available and accumulate normally. Pseudo-rewards are additional local incentives to shape sub-task behaviour — they supplement, not replace, the core reward signal.

Assignment 11 · Q10

If sub-task $M_i$ invokes sub-task $M_j$, do $M_j$'s pseudo-rewards affect $M_i$?

Solution
✓ Yes

$M_j$'s pseudo-rewards are numerically local and not directly added to $M_i$'s return. However, they shape what policy $M_j$ learns. Since $M_i$ relies on $M_j$'s behaviour to achieve its goals, $M_j$'s pseudo-rewards indirectly but fundamentally affect $M_i$'s performance.

Section 05

Option Discovery

Rather than a human manually designing the sub-task hierarchy, Option Discovery algorithms find useful sub-goals automatically from the structure of the state space.

What Makes a Good Option?

Assignment 11 · Q8

Which can be considered a good option in HRL?
(a) Can be reused often. (b) Cuts down exploration. (c) Helps transfer learning.

Solution
✓ All of (a), (b), (c)

(a) "Exit the room" is useful regardless of the ultimate global goal. (b) A macro-action to the doorway eliminates random wandering. (c) In a new but similar environment, the "exit room" option transfers directly without relearning from scratch.

Bottleneck Detection

The most common automatic approach: find bottleneck states — states that must be passed through to reach many different goals. In a state-space graph, bottlenecks are nodes connecting densely-connected regions (like doorways between rooms).

✓ Bottleneck states are perfect natural targets for options — the agent must pass through them regardless of its goal, so an option to reach the bottleneck is universally useful.
Assignment 11 · Q7

Bottlenecks are useful surrogate measures for option discovery. True or False?

Solution
✓ True

In the state-space graph, a bottleneck connects densely-connected regions. The agent statistically must pass through it to reach many different goals, making it an ideal sub-goal for a reusable, exploration-reducing option.
