Value & Policy Iteration

Core algorithms for finding optimal policies and value functions in MDPs: Value Iteration, Policy Iteration, Asynchronous DP, and Monte Carlo methods.

Section 01

Value Iteration & Contraction Mappings

Value iteration successively applies the Bellman optimality operator to an estimated value function $v$ until it converges to the unique optimal value function $v^*$. This works because the operator is a contraction mapping on a Banach space.

The Bellman Optimality Operator

We define operator $L$ applied to a value function $v$ as:

Bellman Optimality Operator
$$(Lv)(s) = \max_{a \in A} \left[ R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)\, v(s') \right]$$

The space of all bounded value functions forms a complete Banach space under the max norm. Key properties of $L$:

  • $L$ is a contraction mapping with factor $\gamma < 1$
  • By the Banach Fixed Point Theorem, $L$ has a unique fixed point $v^*$
  • $L$ is non-linear due to the $\max$ operation

Contraction Bounds

Global Norm Contraction
$$\| Lv - Lu \| \;\leq\; \gamma\, \| v - u \|$$
State-wise Bound
$$\forall s,\quad |Lv(s) - Lu(s)| \;\leq\; \gamma\, \| v - u \|$$
💡 Because $\gamma < 1$, repeated application of $L$ strictly shrinks the distance between any two value functions — guaranteeing convergence to the unique $v^*$.

Stopping Criterion

We stop when the maximum change between successive iterations is small enough:

Stopping Condition
$$\| v^{n+1} - v^n \| < \frac{\epsilon(1 - \gamma)}{2\gamma}$$

This stopping condition guarantees:

  • $\| v^{n+1} - v^* \| \leq \epsilon/2$ (value function is close to optimal)
  • $\| v^\pi - v^* \| \leq \epsilon$ (greedy policy derived from $v^{n+1}$ is near-optimal)
Assignment 5 · Q9

Given that $L$ is a contraction in Banach space, which are true?
(a) A contraction mapping does not have a unique fixed point.
(b) A contraction mapping may be non-linear.
(c) $\|Lv - Lu\| \leq \gamma \|v - u\|$
(d) $\forall s,\; |Lv(s) - Lu(s)| \leq \gamma \|v - u\|$

Solution
✓ (b), (c), (d)

A contraction on a Banach space always has a unique fixed point (so (a) is false). The Bellman operator involves $\max$, making it non-linear (b). The global contraction bound (c) mathematically implies the pointwise bound (d).

Assignment 5 · Q10

(a) It defines a contraction in Banach space.
(b) It can be rewritten as a linear transformation.
(c) Final values at stopping condition are optimal ($v^*$).
(d) Final greedy policy at stopping condition is optimal.

Solution
✓ (a) only

The $\max$ operator makes it non-linear (b false). The stopping criterion only guarantees $\epsilon$-closeness to $v^*$, not exact optimality (c, d false).

Practice Assignment 5 · Q3

Setting the threshold to $\frac{\epsilon(1-\gamma)}{2\gamma}$ is precisely what produces these two guarantees.

Section 02

Policy Iteration

Policy Iteration alternates between two steps: fully evaluating the current policy (Policy Evaluation) and greedily improving it (Policy Improvement). It converges to the exact optimal policy for finite MDPs.

Step 1 — Policy Evaluation (PE)

Compute the value function $v^{\pi_n}$ for current policy $\pi_n$. This is the unique fixed point of the policy Bellman operator $L_\pi$:

Bellman Operator for Policy π
$$(L_\pi v)(s) = \sum_{a} \pi(a \mid s)\left[ R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)\, v(s') \right]$$

PE solves for $v$ such that $L_\pi v = v$. Note that $L_\pi$ is a linear operator (no max), unlike $L$.

Step 2 — Policy Improvement (PI)

Extract a new, better policy by acting greedily with respect to $v^{\pi_n}$:

Greedy Policy Extraction
$$\pi_{n+1}(s) = \arg\max_{a} \left[ R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)\, v^{\pi_n}(s') \right]$$
💡 PE and PI interact monotonically: each PI step guarantees $v^{\pi_{n+1}} \geq v^{\pi_n}$ pointwise. Because there are finitely many deterministic policies, convergence to $v^*$ is guaranteed.
Assignment 5 · Q1

Which are true of the PE and PI steps?
(a) Values returned by PE may fluctuate between high and low values.
(b) PE returns the fixed point of $L_{\pi_n}$.
(c) PI can randomly select any greedy policy for ties.
(d) Policy iteration always converges for a finite MDP.

Solution
✓ (b), (d)

PE values monotonically increase (a false). Ties must be broken consistently to guarantee convergence (c false). PE exactly solves for the fixed point (b true), and finite MDPs have finite deterministic policies so convergence is guaranteed (d true).

Practice Assignment 5 · Q1

After policy iteration converges, which statement is true about $\pi_n$ and $v^{\pi_n}$?
(a) $\pi_n$ optimal but $v^{\pi_n}$ might not be.
(b) $v^{\pi_n}$ optimal but $\pi_n$ might not be.
(c) Nothing can be said.
(d) Both $\pi_n$ and $v^{\pi_n}$ are optimal.

Solution
✓ (d)

When policy iteration's stopping criterion is met (policy stops changing), it has found the exact optimal policy and value function — unlike value iteration which only guarantees $\epsilon$-optimality.

Practice Assignment 5 · Q5

If $v$ satisfies $Lv = L_\pi v = v$, then:

Solution
✓ (c) — $\pi$ is an optimal policy and $v$ is its fixed point

$L_\pi v = v$ means $v = v^\pi$. $Lv = v$ means $v = v^*$. Together, $v^\pi = v^*$, so $\pi$ must be optimal.

Section 03

DP Variants & Generalized Policy Iteration

Classic DP requires sweeping the entire state space and knowing the full environment model $P(s' \mid s, a)$. Variants reduce computational cost while retaining convergence guarantees.

Asynchronous DP

  • Updates states in any order, in place (single value array)
  • Converges as long as every state is updated infinitely often
  • Avoids expensive full sweeps on large state spaces

Real-Time DP (RTDP)

  • Focuses updates along sampled trajectories
  • Saves computation on rarely-visited states
  • Particularly useful in goal-directed tasks

Generalized Policy Iteration (GPI)

GPI is the unifying framework behind all DP and RL methods: policy evaluation and policy improvement interact continuously, without waiting for either to fully complete.

  • No need to wait for full evaluation before improving
  • Before convergence, PE will cause the policy to no longer be greedy
  • System stabilises when $\pi$ is greedy w.r.t. its own evaluated value function
Section 04

Monte Carlo Methods

When the transition model is unknown, DP fails. Monte Carlo methods learn value functions directly from complete episodes of experience, requiring no model at all.

  • Updates are applied only at the end of each episode
  • Works with episodic tasks; learns from sample trajectories
  • Does not bootstrap (uses actual observed returns)
Assignment 5 · Q2

You sample trajectory $S_1 \to S_5 \to S_4 \to S_6 \to \text{terminal}$. Which states can be updated?

Solution
All states: S₁, S₄, S₅, S₆

All states appearing in the trajectory can be updated at episode end using their accumulated returns.

First-Visit vs Every-Visit MC

Method Which visits counted? Bias
First-Visit MC Only the first occurrence per episode Unbiased
Every-Visit MC All occurrences per episode Slightly biased (short-term), asymptotically consistent

Exploration Strategies

  • Exploring Starts: Forces episodes to begin in random state–action pairs
  • ε-greedy: Assigns small probability $\epsilon/|A|$ to all actions — ensures adequate exploration without deterministic starts
Practice Assignment 5 · Q2

Assertion: MC policy evaluation must use exploring starts for non-deterministic policies.
Reason: They must rely on exploring starts to ensure adequate sampling of all states.

Solution
✓ (d) — Both Assertion and Reason are false

Non-deterministic policies inherently explore — their own randomness ensures all state–action pairs are eventually sampled with enough episodes. Exploring starts are not required.

ε-Greedy Policy

ε-Greedy Probabilities
$$P(a^*) = 1 - \epsilon + \frac{\epsilon}{|A|} \qquad \text{(optimal action)}$$ $$P(a) = \frac{\epsilon}{|A|} \qquad \text{(all other actions)}$$
Practice Assignment 5 · Q4

Assertion: Any ε-greedy policy w.r.t. $q_\pi$ is strictly better than $\pi$.
Reason: Optimal action chosen with prob $1 - \epsilon + \epsilon/n$; others with $\epsilon/n$.

Solution
✓ (c) — Assertion false, Reason true

The assertion is false because the ε-greedy policy is at least as good as $\pi$ (equal if already at optimal ε-soft policy). The reason correctly states the ε-greedy probability formula.
