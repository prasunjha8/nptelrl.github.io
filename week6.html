Section 01

Temporal Difference Learning & TD(0)

TD learning is a hybrid of Monte Carlo and Dynamic Programming. Like MC, it learns directly from experience with no model. Like DP, it bootstraps — updating estimates based on other learned estimates without waiting for a final outcome.

💡 The key insight: TD doesn't wait for the episode to end. It updates after every single step using the next state's estimated value.

TD(0) Update Equation

TD(0) State-Value Update
$$V(s_t) \;\leftarrow\; V(s_t) + \alpha \underbrace{\left[R_{t+1} + \gamma V(s_{t+1}) - V(s_t)\right]}_{\delta_t\;\text{(TD error)}}$$

Breaking this down:

  • Target: $R_{t+1} + \gamma V(s_{t+1})$ — uses one real reward + estimate of next state
  • TD Error $\delta$: difference between target and current estimate
  • $\alpha$: learning rate controlling how much we shift toward the target
TD Error (δ)
$$\delta_t = R_{t+1} + \gamma V(s_{t+1}) - V(s_t)$$
Practice Assignment 6 · Q1

In the SARSA update $Q_{new}(s_t,a_t) = Q_{old}(s_t,a_t) + \alpha[R_{t+1} + \gamma Q_{old}(s_{t+1},a_{t+1}) - Q_{old}(s_t,a_t)]$, which part is the TD error $\delta$?
(a) $R_{t+1} + \gamma Q_{old}(s_{t+1},a_{t+1})$
(b) $\alpha[\ldots]$
(c) $R_{t+1} + \gamma Q_{old}(s_{t+1},a_{t+1}) - Q_{old}(s_t,a_t)$
(d) $Q_{old}(s_{t+1},a_{t+1}) - Q_{old}(s_t,a_t)$

Solution
✓ (c)

TD error is the difference between the target and the current estimate, before multiplying by $\alpha$. The $\alpha$ is just the learning rate applied on top.

(ii) Uses a sample of the reward from the distribution of rewards.

Practice Assignment 6 · Q2

Which are true for TD(0) (assuming a truly Markov environment)?
(a) Uses the full return to update values.
(b) TD(0) and MC converge to the same value function given finite samples.
(c) TD(0) and MC converge to the same value function given infinite samples.
(d) TD error is...

Solution
✓ (c)

With infinite samples in a true Markov environment, both converge to the true $v^\pi$. With finite data they diverge: MC minimises MSE on the batch; TD(0) finds the maximum-likelihood Markov model solution.

Worked Numerical Example — TD(0)

MDP with states A, B. Trajectory: A→3→B→2→A→5→B→2→A→4→END. Parameters: $\gamma=1$, $\alpha=1$, $V(A)=V(B)=0$ initially.

With $\alpha=1$, the update simplifies to: $V(s_t) = R_{t+1} + V(s_{t+1})$

Step 1: A →3→ B   →   V(A) = 3 + V(B) = 3 + 0 = 3
Step 2: B →2→ A   →   V(B) = 2 + V(A) = 2 + 3 = 5
Step 3: A →5→ B   →   V(A) = 5 + V(B) = 5 + 5 = 10
Step 4: B →2→ A   →   V(B) = 2 + V(A) = 2 + 10 = 12
Step 5: A →4→ END   →   V(A) = 4 + 0 = 4
Final: V(A) = 4, V(B) = 12
Assignment 6 · Q9

Using the trajectory above with $\gamma=1$, $\alpha=1$, initial values 0. What are final V(A), V(B)?

Solution
✓ (c) — V(A) = 4, V(B) = 12

Comparison: DP vs TD vs MC

Method Backup Type Bootstrapping? Needs Full Model? Updates When?

Temporal Difference Sample (single transition) Yes No Each step
Monte Carlo Sample (full episode) No No End of episode
Assignment 6 · Q2

Which are true?
(a) DP uses full backups and bootstrapping.
(b) TD uses sample backups and bootstrapping.
(c) MC uses sample backups and bootstrapping.
(d) MC uses full backups and no bootstrapping.

Solution
✓ (a), (b)

DP sweeps all transitions (full backup) and uses value estimates (bootstrapping). TD uses single sampled transitions and bootstraps. MC waits for the full return — no bootstrapping (c false, d partially wrong because MC uses sample backups, not full).

Section 03

On-Policy vs Off-Policy Control

SARSA — On-Policy TD Control

SARSA evaluates and improves the same policy generating behavior. Named after the quintuple $(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})$:

SARSA Update
$$Q(S_t, A_t) \;\leftarrow\; Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t) \right]$$

The critical detail: $A_{t+1}$ is the action actually taken by the policy in the next state — hence on-policy.

Practice Assignment 6 · Q4

Assertion: SARSA is an on-policy method.
Reason: In SARSA, we do not update using the action that was actually taken.

Solution
✓ (c) — Assertion true, Reason false

SARSA IS on-policy (assertion correct). The reason is wrong: SARSA updates using $A_{t+1}$, which IS the action actually taken — that's the exact definition of on-policy.

Assignment 6 · Q10

Which are true for SARSA?
(a) It uses bootstrapping. (b) Off-policy. (c) On-policy. (d) Selects actions greedily.

Solution
✓ (a), (c)

SARSA is a TD method (bootstraps ✓). It's on-policy ✓. It uses ε-greedy, not purely greedy, to ensure exploration.

Q-Learning — Off-Policy TD Control

Q-learning learns the optimal policy independently of the behavior policy. It always updates using the best possible next action regardless of what was actually taken:

Q-Learning Update
$$Q(S_t, A_t) \;\leftarrow\; Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma \max_{a'} Q(S_{t+1}, a') - Q(S_t, A_t) \right]$$
💡 The $\max$ operation is what makes Q-learning off-policy. Even while following an ε-greedy behavior policy, it evaluates the greedy target policy.
Assignment 6 · Q5

Assertion: Q-learning can use asynchronous samples from different policies to update Q values.
Reason: Q-learning is an off-policy algorithm.

Solution
✓ (a) — Both true, Reason correctly explains Assertion

Because Q-learning uses the greedy max rather than the actual next action, it's off-policy. This allows it to learn from samples generated by any sufficiently exploratory policy.

SARSA vs Q-Learning: Cliff World

In cliff walking (falling off = −100 reward), with ε-greedy behavior:

AlgorithmPath FoundWhy
Q-Learning Optimal (risky, cliff edge) Evaluates greedy policy — ignores ε-exploration risk
SARSA Safer (away from cliff) Evaluates the ε-greedy policy itself — accounts for random exploration falls
Assignment 6 · Q7

In cliff walking with ε-greedy, which finds the optimal path and which finds the safer path?

Solution
✓ (a) — Q-learning: optimal; SARSA: safer
Section 04

Off-Policy MC & Importance Sampling

When evaluating a target policy $\pi$ using trajectories generated by a behavior policy $q$ (or $\mu$), we must correct for the distribution mismatch using Importance Sampling.

Weighted Importance Sampling Estimator
$$V(s) \;\approx\; \frac{\sum_{i=1}^N f(x_i)\,\frac{\pi(x_i)}{q(x_i)}}{\sum_{i=1}^N \frac{\pi(x_i)}{q(x_i)}}$$

Here $f(x_i)$ is the return on trajectory $x_i$, $\pi(x_i)$ is the trajectory probability under the target policy, $q(x_i)$ under the behavior policy.

Practice Assignment 6 · Q3

In the weighted IS formula, $p$ is _____ and $q$ is _____.
(a) $p$ is estimation (target) policy, $q$ is behaviour policy.
(b) $p$ is behaviour policy, $q$ is estimation policy.

Solution
✓ (a)

The IS weight ratio is always $\frac{\text{target}}{\text{behavior}}$. We're upweighting trajectories the target policy would prefer and downweighting those it wouldn't.

Assignment 6 · Q1

Assertion: Off-policy MC importance sampling requires knowledge of MDP transition probabilities.
Reason: Both policies' trajectory probabilities depend on transition probabilities.

Solution
✓ (d) — Both Assertion and Reason are false

The transition probabilities $P(s' \mid s,a)$ appear in both numerator and denominator of the IS ratio — they cancel out perfectly. We only need the policies' action probabilities, not the environment model.

Section 05

Planning vs Learning

ConceptSource of UpdatesModel Required?
Learning Real-world interactions (sample experience) No — model-free
Planning Simulated model of the MDP Yes — at least a simulation model
Practice Assignment 6 · Q5

Which are true?
(a) Learning requires at least a simulation model.
(b) Planning requires at least a simulation model.
(c) Learning can be done just using real-world samples.
(d) Planning can be done just using real-world samples.

Solution
✓ (b), (c)

Learning is model-free and works from real experience (c). Planning, by definition, needs a model to simulate transitions (b).

Assignment 6 · Q8

Primary advantage of a simulated model over real-world experience?

Solution
✓ (c) — Safe exploration of risky states

A simulator lets the agent try catastrophic actions (cliff falls, crashes) without real-world consequences — invaluable for robotics, safety-critical systems.

Assignment 6 · Q3

Key characteristic of UC-Trees algorithm?

Solution
✓ (c) — Balances exploration and exploitation via confidence bounds

UCT (Upper Confidence Trees) applies UCB formulas to Monte Carlo Tree Search, mathematically balancing known-good actions with uncertain ones.

Assignment 6 · Q6

What is an after-state in RL?

Solution
✓ (b) — State immediately after action, before environment's transition

In deterministic games (e.g., tic-tac-toe), the state after your move is deterministic even if the opponent's response isn't. Evaluating after-states is more efficient since multiple state-action pairs can lead to the same after-state.
