n-step & Eligibility Traces

Section 01

n-step Bootstrapping

TD(0) looks exactly 1 step ahead; Monte Carlo looks all the way to the episode end. n-step methods generalise by waiting exactly $n$ real steps before bootstrapping:

The n-step Return

The n-step truncated return looks $n$ real steps into the future, then bootstraps with the current value estimate:

n-step Truncated Return
$$G_t^{(n)} = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots + \gamma^{n-1} R_{t+n} + \gamma^n V_t(s_{t+n})$$
Assignment 7 · Q1

Which is the correct n-step truncated return?
(a) $R_{t+n} + \gamma^n V_t(s_{t+n})$
(b) $R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{n-1}R_{t+n} + \gamma^n V_t(s_{t+n})$
(c) $\gamma R_{t+1} + \gamma^2 R_{t+2} + \cdots + \gamma^n R_{t+n} + V_t(s_{t+n})$

Solution
✓ (b)

Rewards are discounted starting from $R_{t+1}$ with $\gamma^0 = 1$, up to $R_{t+n}$ with $\gamma^{n-1}$, then the value estimate gets $\gamma^n$.

Practice Assignment 7 · Q1

Which equals $G_t^{(3)}$?
(a) $R_{t+1} + \gamma R_{t+2} + \gamma^2 V(s_{t+2})$
(b) $R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \gamma^3 V(s_{t+3})$
(c) $R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \gamma^3 V(s_{t+4})$

Solution
✓ (b)

Substitute $n=3$: three real rewards ($R_{t+1}, R_{t+2}, R_{t+3}$) with discounts $1, \gamma, \gamma^2$, then $\gamma^3 V(s_{t+3})$.

Section 02

Eligibility Traces & TD(λ)

Rather than picking a fixed $n$, Eligibility Traces elegantly implement a weighted average of all n-step returns at once. Every state has a trace $E_t(s)$ tracking how "eligible" it is for credit.

Forward View vs Backward View

ViewHow it worksPractical?
Forward View Target $G_t^\lambda$ is weighted average of all n-step returns. Conceptually elegant. No — requires waiting until end of episode
Backward View Maintain trace $E_t(s)$ per state; update all states at every step using current TD error. Yes — fully online, equivalent to forward view
💡 The trace $E_t(s)$ acts as a "recency-weighted memory" — states visited recently have high eligibility and receive more credit when a TD error arrives.

λ-Return: Bridging TD and MC

The $\lambda$-return is a geometrically-weighted average of all n-step returns:

Connection to λ=0 and λ=1
$$\lambda = 0 \implies TD(0) \qquad \lambda = 1 \implies \text{Monte Carlo}$$
Practice Assignment 7 · Q3

Is TD(λ) with $\lambda=1$ essentially Monte Carlo?

Solution
✓ Yes

With $\lambda=1$, no decay — every state gets full credit for all future rewards until terminal, exactly the MC return $G_t$.

Assignment 7 · Q3

Assertion: TD(1) extends MC for continuing tasks.
Reason: TD(1) with eligibility traces is an offline algorithm.

Solution
✓ (c) — Assertion true, Reason false

TD(1) achieves the MC return but applies updates online at each step via the trace mechanism — unlike true MC which is fully offline. This lets TD(1) work on continuing (non-episodic) tasks.

Assignment 7 · Q6

Assertion: Forward view trace target can be used for mid-episode updates.
Reason: $G_t^\lambda$ is computable after a finite number of steps in all cases.

Solution
✓ (d) — Both false

The forward view target requires knowing all future n-step returns — impossible until the episode ends. Both the assertion and the reason are false.

Assignment 7 · Q5

For $\lambda \in (0,1)$, the one-step return always gets the maximum weight in the λ-return. True or false?

Solution
✓ False

Weight depends on both $\lambda$ and episode length. For very short episodes with high $\lambda$, a later return can carry more total normalised weight.

Section 03

Accumulating vs Replacing Traces

When a state is visited, how do we update its trace?

TypeOn visit to state sOtherwiseIssue
Accumulating $E_t(s) = \gamma\lambda E_{t-1}(s) + 1$ $E_t(s) = \gamma\lambda E_{t-1}(s)$ Trace can exceed 1 → over-updating
Replacing $E_t(s) = 1$ $E_t(s) = \gamma\lambda E_{t-1}(s)$ None — capped at 1, often better empirically
Replacing Trace Update Rule
$$E_t(s) = \begin{cases} 1 & \text{if } s_t = s \\ \gamma\lambda\, E_{t-1}(s) & \text{otherwise} \end{cases}$$
Assignment 7 · Q2

Which is the correct way to implement replacing traces?

Solution
✓ (d) — The formula above

Simply reset to 1 if currently visiting that state; otherwise decay by $\gamma\lambda$ as usual.

Section 04

SARSA(λ)

SARSA(λ) extends SARSA by applying traces to state-action pairs $E_t(s,a)$, not just states. This distributes credit backward through the entire trajectory of $(s,a)$ pairs.

Practice Assignment 7 · Q2

SARSA(λ) applies TD(λ) prediction to just the states rather than state-action pairs. True or false?

Solution
✓ False

Control algorithms must evaluate actions to form policies. SARSA(λ) maintains traces for $(s,a)$ pairs and updates $Q(s,a)$ directly.

Aggressive Replacing Traces for SARSA(λ)

When visiting state $s$ and taking action $a_1$, we must zero out traces for all other actions in $s$:

SARSA(λ) Replacing Trace — State-Action
$$E_t(s,a) = \begin{cases} 1 & \text{if } s_t = s,\; a_t = a \\ 0 & \text{if } s_t = s,\; a_t \neq a \\ \gamma\lambda\, E_{t-1}(s,a) & \text{otherwise} \end{cases}$$
Assignment 7 · Q9

Why do we set $E_t(s,a) = 0$ for all $a \neq a_1$ when state $s$ is visited with action $a_1$?
(i) Rewards from $a_1$ shouldn't be attributed to previous actions in $s$.
(ii) Assumed time between visits is large enough to decay trace to 0.

Solution
✓ Only (i)

If you take $a_1$ in $s$, the subsequent reward causally comes from $a_1$. Allowing old actions' traces to remain would give them false credit for $a_1$'s outcome.

Assignment 7 · Q7

Which is the correct replacing trace update for SARSA(λ) to only update the estimate for action $a$ taken in state $s$?

Solution
✓ (d) — The three-case formula above

Match → set to 1. Same state, different action → zero. Different state → decay normally by $\gamma\lambda$.

Section 05

Watkins's Q(λ)

Watkins's Q(λ) combines Q-learning with eligibility traces. Because Q-learning targets the optimal (greedy) policy, traces can only safely accumulate while the agent actually takes the greedy action.

⚠️ The moment an exploratory (non-greedy) action is taken, all traces are cut to zero. Subsequent experience no longer represents the optimal policy so backward credit assignment would be misleading.
Assignment 7 · Q8

Assertion: With ε-greedy exploration and Watkins Q(λ), ε must be kept low.
Reason: High ε causes traces to be too short, negating eligibility trace advantages.

Solution
✓ (a) — Both true, Reason correctly explains

Every exploratory action zeroes all traces. High ε means frequent cuts → traces never grow long → algorithm degrades to plain 1-step Q-learning.

Practice Assignment 7 · Q4

In Q(λ), if the first action is exploratory (non-optimal), is a TD update still made for that transition?

Solution
✓ Yes

A 1-step Q-learning update is always made for the actual transition. Only the traces of previous states are cut — the immediate update still happens.

Section 06

Off-Policy Learning with Traces

For off-policy TD(λ), the importance sampling ratio $\rho_t = \frac{\pi(s_t,a_t)}{\mu(s_t,a_t)}$ must be folded into every trace update:

Off-Policy Trace Update
$$E_t(s) = \begin{cases} \rho_t\left(\gamma\lambda\, E_{t-1}(s) + 1\right) & \text{if } s_t = s \\ \rho_t\,\gamma\lambda\, E_{t-1}(s) & \text{otherwise} \end{cases}$$
💡 If the behavior policy diverges heavily from the target, $\rho_t \to 0$, which naturally cuts the trace — the same effect as an exploratory action in Q(λ), but automatic.
Assignment 7 · Q10

In off-policy TD(λ), which is the correct trace update given importance ratio $\rho_t$?

Solution
✓ (b) — The formula above

Multiply the entire updated trace by $\rho_t$ at every step to properly weight the credit assignment by how likely the target policy was to take the observed action.

Assignment 7 · Q4

Assertion: The behavior policy need not have non-zero probability for all actions.
Reason: If estimation policy has zero probability for an action, behavior policy can too without causing problems.

Solution
✓ (a) — Both true, Reason correctly explains

The behavior policy only needs coverage for actions the target policy might take. If the target policy assigns zero probability to an action, it doesn't matter if behavior policy also skips it — the IS ratio handles this correctly.
