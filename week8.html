Section 01

Why Approximation? The Curse of Dimensionality

In Weeks 5–7, every state $s$ had its own dedicated table entry $V(s)$. This works perfectly for small grid worlds but collapses in real problems for two reasons:

  • Massive discrete spaces: Backgammon has ~$10^{20}$ states. At 1 byte each, no storage system on Earth suffices.
  • Continuous spaces: Robot joint angles are continuous — there are literally infinitely many states. Without discretisation, a lookup table is mathematically impossible.

The solution is a parameterised function $\hat{v}(s, \mathbf{w})$ with a small weight vector $\mathbf{w}$. Visiting state $s_1$ and learning its value generalises automatically to similar, unvisited states.

Assignment 8 · Q4

Solution

A lookup table requires a discrete key for each entry. Continuous spaces have infinite points — no finite table can map them 1-to-1.

Section 02

Linear Function Approximation & Gradient Descent

The Linear Model

Extract $k$ numerical features from state $s$ into a vector $\phi(s) = [\phi_1(s), \dots, \phi_k(s)]^\top$. The value estimate is:

Linear Value Approximation
$$\hat{v}(s, \mathbf{w}) = \mathbf{w}^\top \phi(s) = \sum_{i=1}^k w_i \phi_i(s)$$

This is classical linear regression: the model is linear in the weights $\mathbf{w}$ (even if the features $\phi$ are non-linear functions of $s$).

Objective: Mean Squared Value Error (MSVE)

We can't minimise error equally across all states — in a huge state space, the agent spends nearly all time in a tiny fraction of states. Fitting irrelevant states wastes the model's capacity.

We weight each state's error by $d^\pi(s)$ — the steady-state distribution, i.e. the fraction of time the agent spends in $s$ under policy $\pi$:

Mean Squared Value Error
$$\text{MSVE}(\mathbf{w}) = \sum_{s \in S} d^\pi(s) \left[ v_\pi(s) - \hat{v}(s, \mathbf{w}) \right]^2$$
💡 By letting the agent act on-policy, the training data naturally reflects $d^\pi(s)$. No explicit calculation needed — the distribution emerges from the agent's own behaviour.
Assignment 8 · Q3

When would the loss $\sum_{s\in S}(\hat{V}(s)-V(s))^2$ (without $d^\pi$ weighting) lead to poor performance?

Solution
✓ (a) — Large state space with a small percentage of relevant states

Without $d^\pi(s)$ weighting, the model tries to fit millions of irrelevant states equally, sacrificing accuracy on the few frequently-visited states that actually matter for performance.

Assignment 8 · Q8

Solution

Linear Regression: Unique Solution

If we have a dataset with feature matrix $\Phi$ and targets $y$, the optimal weights are the closed-form solution:

Normal Equation
$$\mathbf{w}^* = (\Phi^\top \Phi)^{-1} \Phi^\top y$$
Assignment 8 · Q1

If column vectors of $\Phi$ are linearly independent, does a unique linear minimiser of MSE exist?

Solution
✓ True

Linear independence of columns ensures $(\Phi^\top\Phi)$ is invertible, giving the unique closed-form solution $\mathbf{w}^* = (\Phi^\top\Phi)^{-1}\Phi^\top y$.

Eligibility Traces with Linear VFA

In tabular $TD(\lambda)$, the trace tracks recently visited states. With VFA, we instead track an eligibility trace vector $\vec{e}_t$ in weight space:

Trace Update — Linear VFA
$$\vec{e}_t = \gamma\lambda\,\vec{e}_{t-1} + \nabla_{\mathbf{w}}\hat{v}(s_t, \mathbf{w})$$

For a linear approximator $\hat{v}(s,\mathbf{w}) = \mathbf{w}^\top\phi(s)$, the gradient is simply the feature vector:

Gradient of Linear Model
$$\nabla_{\mathbf{w}}\hat{v}(s_t, \mathbf{w}) = \phi(s_t)$$
Practice Assignment 8 · Q4

Correct update equation for eligibility traces with linear VFA $V(s_t) = \mathbf{w}^\top\phi(s_t)$?

Solution
✓ (b) — $\vec{e}_t = \gamma\lambda\,\vec{e}_{t-1} + \phi(s_t)$

The gradient of a linear model is its feature vector. We decay past traces by $\gamma\lambda$ and add the current state's features.

Section 03

The Moving Target Problem

In supervised learning, labels are fixed. In TD learning, the target is bootstrapped — it depends on our own current weights:

TD Target (Non-Stationary)
$$y_t = R_{t+1} + \gamma\,\hat{v}(s_{t+1}, \mathbf{w})$$
⚠️ When we update $\mathbf{w}$ to reduce error on $s_t$, the target $y_t$ for every other state also changes. We are chasing a moving target.

The consequence: TD with function approximation converges to the TD Fixed Point, which is provably different (and slightly worse) than the true MSVE-minimising solution you'd get with exact Monte Carlo targets.

Assignment 8 · Q5

Assertion: Gradient descent on MSE between $\hat{v}(s_t)$ and bootstrapped targets $R_{t+1} + \gamma\hat{v}(s_{t+1})$ will converge to the same solution as using true $v_\pi$ targets.
Reason: Each update moves $\mathbf{w}$ closer to $v_\pi$, so bootstrapped targets eventually converge to true values.

Solution
✓ (d) — Both Assertion and Reason are false

Bootstrapping means the system converges to the TD Fixed Point, which is mathematically distinct from the exact MSVE solution. The reason is also wrong — there is no monotonic convergence guarantee to $v_\pi$.

Section 04

Feature Construction: Coarse Coding & Tile Coding

Linear approximation is only as powerful as its features. Raw continuous coordinates only allow a linear decision boundary — too simple for most tasks. We need non-linear features.

Coarse Coding

Scatter overlapping receptive fields (circles/ellipses) over the continuous state space. Each state $s$ is represented as a binary feature vector:

  • $\phi_i(s) = 1$ if state $s$ falls inside circle $i$
  • $\phi_i(s) = 0$ otherwise

Because circles overlap, learning the value of one state updates the weights shared with nearby states — creating smooth, local generalisation.

Tile Coding

A structured, efficient form of coarse coding. Instead of random circles, overlay multiple offset grids (tilings) over the state space. Each state activates exactly one tile per tiling.

  • With $n$ tilings: every state activates exactly $n$ binary features
  • Offsets between grids create overlap → smooth generalisation
  • Works for continuous spaces — it discretises them into overlapping bins
  • Computationally very efficient (hash-based lookup)
Assignment 8 · Q9

Solution

Section 05

Control (Action Selection) with VFA

Estimating $V(s)$ is insufficient for control without a model — you can't do a one-step lookahead if you don't know $P(s' \mid s,a)$. You must approximate action-values:

Action-Value Approximation
$$\hat{q}(s, a, \mathbf{w}) \approx Q(s, a)$$

Discrete Action Spaces

With finitely many actions (e.g., Up/Down/Left/Right), greedy action selection is trivial:

  • Evaluate $\hat{q}(s, a, \mathbf{w})$ for each possible action $a$
  • Pick $\arg\max_a \hat{q}(s, a, \mathbf{w})$
  • No classifier needed — just compute $k$ dot products and take the max
Assignment 8 · Q6

Assertion: To solve $\pi(s) = \arg\max_a Q(s,a)$ with a discrete action space and linear function approximator, we need to formulate a classification problem.

Solution
✓ (d) — Both Assertion and Reason are false

No classifier needed. With discrete actions, evaluate the linear function for each action and take the numerical maximum — straightforward and cheap.

Continuous Action Spaces

If actions are continuous (e.g., steering angle $\in [0, 1]$), you cannot enumerate all actions. Instead, a regressor directly outputs the optimal continuous action from the state.

Practice Assignment 8 · Q2

Assertion: LSPI (Least Squares Policy Iteration) can be used for continuous action spaces.
Reason: We can train a regressor on (state, action) pairs to approximate the policy.

Solution
✓ (a) — Both true, Reason correctly explains

By parameterising the policy and fitting it via regression on sampled data, LSPI outputs continuous actions directly.

Section 06

Least Squares TD Methods (LSTD, LSTDQ, LSPI)

Standard SGD-based TD uses data once and discards it — sample-inefficient. Batch methods accumulate all collected data and solve for optimal weights in one shot using linear algebra.

MethodEstimatesAlgorithm TypeReuses Data?
LSTD $V(s)$ (state values) Batch matrix inversion Yes
LSTDQ $Q(s,a)$ (action values) Batch matrix inversion Yes
LSPI Policy Policy iteration using LSTDQ Yes

The LSTD Linear System

Instead of iterating gradient steps, LSTD sets the expected update to zero and solves the resulting linear system $A\mathbf{w} = b$ directly:

LSTD Closed-Form Solution
$$\mathbf{w} = \tilde{A}^{-1}\tilde{b}$$

$\tilde{A}$ and $\tilde{b}$ are estimated from sampled transitions $(s_t, r_{t+1}, s_{t+1})$, and built incrementally — you can solve at any point mid-trajectory by inverting the current $\tilde{A}$.

✓ LSTD key advantages: no learning rate to tune, reuses every sample multiple times, converges in far fewer samples than online SGD. Works only with linear function approximation.
Assignment 8 · Q7

Which are true about LSTD and LSTDQ?
(a) Both are iterative gradient descent algorithms.
(b) Both can reuse samples.
(c) Both require linear function approximation.

Solution
✓ (b), (c)

(a) False — these are batch matrix inversion algorithms, not iterative gradient descent. (b) Yes — all collected data is used simultaneously. (c) Yes — the formulation requires the linear structure to set up the matrix equations.

Practice Assignment 8 · Q3

Can $\tilde{A}$ and $\tilde{b}$ be built incrementally so we can stop and solve at any point in a trajectory?

Solution
✓ True

The matrices are sums of outer products of feature vectors. Each new transition simply adds to the running sum. No raw data needs to be stored — invert the current $\tilde{A}$ at any time to get the best weights for all data seen so far.

LSTDQ Matrix Formulation

LSTDQ extends LSTD to $Q$-values. The matrix $\tilde{A}$ captures the expected TD-update structure:

LSTDQ — Estimated Matrix Ã
$$\tilde{A} = \frac{1}{L}\sum_{i=1}^{L} \left[ \phi(s_i, a_i)\left(\phi(s_i, a_i) - \gamma\,\phi(s'_i, \pi(s'_i))\right)^\top \right]$$

Here $\phi(s_i, a_i)$ is the feature vector for the current state-action, and $\phi(s'_i, \pi(s'_i))$ is for the next state paired with the action the target policy $\pi$ would take — exactly the bootstrapping structure of Q-learning.

Assignment 8 · Q10

Correct values for $\tilde{A}$ in LSTDQ, given samples $D = \{s_i, a_i, s'_i, r_i\}$?

Solution
✓ (d) — The formula above

Current features are $\phi(s_i, a_i)$. The target features use $\phi(s'_i, \pi(s'_i))$ — the next state combined with the policy's chosen action, scaled by $\gamma$. The outer product captures the "expected improvement direction" across the dataset.
